1. الرؤية والهوية

ممهور هي منصة عربية متكاملة لإدارة دورة حياة المحتوى واعتماده. الاسم مشتق من "ختم" (مَمْهور = مُختَم بالختم الرسمي) — ما يعكس الفلسفة الأساسية: كل محتوى يمر بمراحل مراجعة واعتماد حتى يصبح "ممهوراً".

3.4 عميل Prisma

يستخدم المشروع Prisma 7 مع adapter-better-sqlite3 — وهو متطلب إلزامي في Prisma 7 حيث تم استبدال محرك الاستعلامات الثنائي بمحركات JavaScript. يتم تهيئة العميل كـ Singleton عبر global.prisma لتجنب إعادة الإنشاء في بيئة التطوير.

7.3 المرحلة 3 — الاستخدام اليومي

رفع مادة جديدة

الخطوةالوصف
1ينتقل إلى /dashboard/content/new
2يختار نوع المادة (نص / صورة / فيديو / صوت)
3يختار المشروع من القائمة المنسدلة (مشاريع حقيقية من DB)
4يملأ العنوان والوصف والمحتوى
5ينقر "رفع المادة"
6createMaterial() — Zod validation → ownership check → material.create() → stateTransition.create(→DRAFT)
7يتم التوجيه لـ /dashboard/content/{id}

دورة حياة المادة

الحالةمن يغيرهاالإجراء
DRAFT → IN_REVIEWالمنشئإرسال للمراجعة
IN_REVIEW → CHANGES_REQUESTEDالمراجعطلب تعديلات مع تعليقات
CHANGES_REQUESTED → IN_REVIEWالمنشئتعديل وإعادة إرسال
IN_REVIEW → PENDING_APPROVALالمراجعتمرير للاعتماد
PENDING_APPROVAL → APPROVEDالمعتمد✓ ممهور!
APPROVED → PUBLISHEDالمالك/المديرنشر
💡 نصيحة كل تحول في الحالة يُسجَّل في جدول StateTransition مع fromStatus, toStatus, userId, note, و createdAt — مما يوفر سجل تدقيق كامل.

12. البنية التحتية للحاويات (Docker)

يوجد ملف docker-compose.yml جاهز لبيئة الإنتاج مع PostgreSQL 15:

services:
  db:
    image: postgres:15
    environment:
      POSTGRES_USER: mamhoor
      POSTGRES_PASSWORD: password
      POSTGRES_DB: mamhoordb
    ports: ["5432:5432"]
    volumes: [postgres_data:/var/lib/postgresql/data]
📝 ملاحظة حالياً غير مُفعّل — المشروع يستخدم SQLite. عند الانتقال لـ PostgreSQL:
  1. تغيير provider في schema.prisma من sqlite إلى postgresql
  2. تحديث DATABASE_URL في .env
  3. استبدال @prisma/adapter-better-sqlite3 بـ @prisma/adapter-pg
  4. تشغيل npx prisma migrate dev

13. التوصيات والخطوات المستقبلية

أولويات قصيرة المدى

  • ربط لوحة كانبان بالبيانات الحقيقية مع Drag & Drop
  • ربط التعليقات بـ Server Action (إضافة/حل)
  • ربط الفريق بـ ProjectMember + دعوة أعضاء
  • إنشاء Server Action لـ مشروع جديد من الداشبورد
  • ربط الإشعارات بأحداث النظام الحقيقية

أولويات متوسطة المدى

  • إضافة رفع الملفات (S3/Cloud Storage) للصور والفيديو والصوت
  • Server Actions لـ تغيير حالة المادة (مراجعة → اعتماد → نشر)
  • تفعيل تعديل الملف الشخصي في الإعدادات
  • إضافة نظام الإشعارات الفورية (WebSocket أو SSE)
  • إضافة البحث في المواد والمشاريع

أولويات طويلة المدى

  • الانتقال من SQLite إلى PostgreSQL للإنتاج
  • إضافة اختبارات (Unit + Integration + E2E)
  • إعداد CI/CD Pipeline
  • إضافة دعم متعدد اللغات (i18n)
  • إضافة سجل التدقيق (Audit Log) المرئي
  • إعداد نظام النسخ الاحتياطي
  • تحسين الأداء — ISR, caching, connection pooling
